# 📱 모바일 테스트 가이드

## 🎯 3가지 테스트 방법

---

## 방법 1: GitHub Pages (추천 ✨)

### 설정 방법:
1. **GitHub 접속**: https://github.com/ohby88/raccoon-game
2. **Settings** 탭 클릭
3. 왼쪽 메뉴에서 **Pages** 클릭
4. **Source** 설정:
   - Branch: `main`
   - Folder: `/ (root)`
5. **Save** 버튼 클릭
6. **5-10분 대기**

### 접속 URL:
```
https://ohby88.github.io/raccoon-game/
```

### 장점:
- ✅ 영구적인 URL
- ✅ 어디서나 접속 가능
- ✅ HTTPS 자동 지원
- ✅ 무료

---

## 방법 2: 로컬 네트워크 (빠름 🚀)

**조건**: PC와 핸드폰이 **같은 Wi-Fi**에 연결

### PC에서:
1. 서버가 실행 중인지 확인
   ```
   http://localhost:8000
   ```

2. PC의 로컬 IP 확인
   ```
   현재 IP: 192.168.219.100
   ```

### 핸드폰에서:
```
http://192.168.219.100:8000
```

### 문제 해결:
- ❌ 연결 안 되면: PC 방화벽 설정 확인
- ❌ 서버 꺼짐: Python 서버 재시작

---

## 방법 3: Vercel 배포 (즉시 🔥)

### 1회 설정 (5분):

1. **Vercel 계정 만들기**:
   - https://vercel.com
   - GitHub로 로그인

2. **프로젝트 임포트**:
   - "New Project" 클릭
   - `ohby88/raccoon-game` 선택
   - "Deploy" 클릭

3. **배포 완료**:
   ```
   https://raccoon-game-xxx.vercel.app
   ```

### 장점:
- ✅ 즉시 배포 (2분)
- ✅ 자동 HTTPS
- ✅ Git push 시 자동 업데이트
- ✅ 무료

## 🐛 디버깅 (브라우저 개발자 도구)

### 모바일 Chrome:
1. PC Chrome에서 `chrome://inspect` 접속
2. USB로 핸드폰 연결 (USB 디버깅 활성화)
3. 핸드폰 화면의 콘솔 로그 확인

### Safari (iOS):
1. iPhone 설정 > Safari > 고급 > 웹 속성 검사기 활성화
2. Mac에서 Safari > 개발 > [기기 이름] > 페이지 선택

### 콘솔에서 확인할 메시지:
```
🔽 플랫폼 통과 활성화! fallThrough: 20
✅ 플랫폼 통과! y: XXX fallThrough: XX
```
